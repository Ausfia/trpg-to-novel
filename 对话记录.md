```python
    }]

```

需要：

- OpenAI 付费账号（绑海外信用卡或使用虚拟卡）

### ✅ 途径 2：OpenRouter（聚合商，支付门槛低）

```python
# SDK 兼容 OpenAI 格式，只需改 base_url 和 api_key
from openai import OpenAI
client = OpenAI(
    base_url="https://openrouter.ai/api/v1",
    api_key="sk-or-v1-xxx"  # 在 openrouter.ai 免费注册获取

response = client.chat.completions.create(
    model="openai/gpt-4o",
    messages=[{"role": "user", "content": "画一只猫"}]
)
```

优势：

- 支持 **加密货币、PayPal、国内部分虚拟信用卡** 充值

- 提供图像生成模型路由

- 域名经常变、随时跑路

## 什么是反向代理（反代）？

## 方案一：Cloudflare Workers（零成本，最推荐）

这是最流行的方案——利用 Cloudflare 的全球边缘网络做中转，**完全免费**，每天 10 万次请求免费额度，个人使用绰绰有余 [3]。

### 步骤

**① 准备工作**

- 一个 Cloudflare 账户（注册：dash.cloudflare.com）
- 一个**未被墙的域名**，托管在 Cloudflare 上（可以是 `.xyz` 之类的便宜域名，几块钱一年）

> ⚠️ Cloudflare 默认的 `workers.dev` 域名在国内已被墙，所以必须绑定自己的域名 [2]。

**② 创建 Worker**

1. 登录 Cloudflare 控制台 → 左侧菜单点击 **Workers 和 Pages**
2. 点击 **创建应用程序** → **创建 Worker**
3. 给你的 Worker 起个名字（比如 `openrouter-proxy`）
4. 点击 **编辑代码**，把下面的代码粘贴进去：

```javascript
// 反代 OpenRouter（也兼容 OpenAI 格式）
const TELEGRAPH_URL = 'https://openrouter.ai';

addEventListener('fetch', event => {
  event.respondWith(handleRequest(event.request))
})

async function handleRequest(request) {
  const url = new URL(request.url);
  // 保持原始路径
  url.host = TELEGRAPH_URL.replace(/^https?:\/\//, '');
  
  const modifiedRequest = new Request(url.toString(), {
    headers: request.headers,
    method: request.method,
    body: request.body,
    redirect: 'follow'
  });
  
  const response = await fetch(modifiedRequest);
  const modifiedResponse = new Response(response.body, response);
  
  // 允许跨域
  modifiedResponse.headers.set('Access-Control-Allow-Origin', '*');
  return modifiedResponse;
}
```

5. 点击 **部署**。

**③ 绑定域名**

1. 在 Worker 详情页 → **触发器** → **自定义域**
2. 添加你的域名，比如 `api.你的域名.xyz`
3. Cloudflare 会自动配置 DNS 和 SSL 证书

**④ 在 Cherry Studio 中使用**

把 API 地址从：

```
https://openrouter.ai/api/v1
```

改成：

```
https://api.你的域名.xyz/api/v1
```

API Key 不变。

```nginx
location ^~ / {
```

## 为什么不能直接运行

| Go 语言编写                  | 只支持 JS/TS/WebAssembly   |

| 监听端口、管理后台 Web UI    | 无法运行 Web 服务器        |

简单说：One API 是一个需要**24 小时运行的后端服务**，而 Cloudflare Workers 是**按需执行的轻量函数**，两者架构完全不同。

## 但可以这么用：Workers 当代理

如果你已经有了 One API 部署（比如在 VPS 上），Workers 可以充当**入口网关**：

```

```

这样做的好处：

- 隐藏 One API 的真实 IP

## 打开即用：Worker 反代 OpenRouter

在 Cloudflare Workers 里新建一个 Worker，把下面代码全部粘贴进去，改一处就行 [1]：

```js
export default {
  async fetch(request) {
    // ========== 这里改成你的 OpenRouter API Key ==========
    const OPENROUTER_KEY = 'sk-or-v1-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx';
    
    const url = new URL(request.url);
    
    // 目标：转发到 OpenRouter
    const targetUrl = 'https://openrouter.ai' + url.pathname + url.search;
    
    // 构造新请求，带上你的 Key
    const newHeaders = new Headers(request.headers);
    newHeaders.set('Authorization', `Bearer ${OPENROUTER_KEY}`);
    newHeaders.set('Host', 'openrouter.ai');
    
    const newRequest = new Request(targetUrl, {
      method: request.method,
      headers: newHeaders,
      body: request.method !== 'GET' && request.method !== 'HEAD' ? request.body : null,
      redirect: 'follow'
    });
    
    let response = await fetch(newRequest);
    
    // 处理 CORS，让任何客户端都能调
    response = new Response(response.body, response);
    response.headers.set('Access-Control-Allow-Origin', '*');
    response.headers.set('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
    response.headers.set('Access-Control-Allow-Headers', '*');
    
    return response;
  }
};
```

图片生成慢，这是正常的，跟你设置的 Worker 代理关系不大。原因出在整个链路和生图模型本身的特性上。

## 核心原因：生图不像对话，没法"边想边出字"

你平时聊天用 `deepseek-chat`，开了流式输出（Stream），每生成一个 token 就实时推送到 Cherry Studio——**字是一个一个往外蹦的**，体验上瞬间有反应。

但生图模型（DALL-E 3）**完全不支持流式输出**[3]。它必须把整张图在后台全部算完，才一次性返回。这个过程叫做**扩散生成**（Diffusion），模型要从纯噪声一步步"去噪"成清晰画面，通常需要：

- **DALL-E 3 standard 质量**：5~15 秒
- **DALL-E 3 HD 质量**：10~30 秒甚至更长[2][3]

在 Cherry Studio 里看到的就是：点了发送 → 界面卡住不动 → 等很久 → 图片突然出现。**这不是卡了，是它在后台算图。**

---

## 你的链路额外增加了哪些延迟？

```
Cherry Studio → Cloudflare Worker → OpenRouter → OpenAI 服务器 → 原路返回
```

| 环节                   | 延迟来源                                    | 影响                 |
| ---------------------- | ------------------------------------------- | -------------------- |
| **OpenAI 推理**        | DALL-E 3 扩散生成                           | **最大头**，5~30 秒  |
| **OpenRouter 中转**    | 路由调度 + 等待模型空闲                     | 额外 1~3 秒[1]       |
| **你的 Worker**        | Cloudflare 全透传转发                       | 几乎可忽略（<100ms） |
| **图片下载**           | DALL-E 3 返回的是未压缩 PNG，通常 2~8 MB[3] | 下载本身可能 1~3 秒  |
| **Cherry Studio 渲染** | 接收完整图片后显示                          | <1 秒                |

**结论：你的 Worker 几乎没有增加任何显著延迟**，慢主要是 DALL-E 3 本身慢 + OpenRouter 额外一跳。
